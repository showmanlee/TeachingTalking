카드군을 나누어 특정 카드군만 뿌릴 수 있도록 한다
	어떤 카드군으로?
		탈것, 먹을 것, 입을 것, 할 수 있는 것, 형용사, 부사, 인칭대명사?
		아무튼 카드군만 추가되면 확장될 수 있도록.
내부적인 데이터 구분 역시 카드군대로 적용(i.e. 탈것 가방의 1번 카드 = 비행기)
실제 낱말카드 역시 분야별로 나누어져있으니까.
우선은 대표 카드 10개씩만 모아보자.

단어 - 카드 하나 집기(하나의 이름은 무엇인가요)
구문 - 카드 여러 개 집기(시스템적으로 단어 꾸미기로 인식)
문장 - 카드 여러 개 집기(시스템적으로 문장으로 인식)
테스트 - 카드 여러 개 집기(말해보세요)

어쨌든 단어 단계에서는 기본적으로 이게 무엇인지를 알려준다(사전처럼)
그리고 단어 조합에 따른 변화는 구문/문장에서 설명한다.

NounCanvas에서 카드는 하나만 선택할 수 있다.
SyntaxCanvas에서는 여러 개의 카드를 선택할 수 있지만, 내부적으로 단어로만 인식하게 한다.
SentenceCanvas에서는 여러 개의 카드를 선택하고, 내부적으로 문장이 된다고 인식하게 한다.
TestCanvas에서 카드를 놓으면 말할 수 있게...

*카드에서 저장하는 것들 - 카드 번호(1-1), 이름, 그 변화 형태들(어떤 카드군과 어떻게 결합이 됐고, 어떻게 바뀌었는가)*
카드 조합은 순서에 영향을 받는다 - '파랑', '자동차'가 있을 때 '파랑' '자동차' 순으로 되면 "파란 자동차", 반대면 "자동차가 파랗다"
카드와 결합됐을 때 카드군에 맞춰 변화 형태를 알고 있다면 그렇게 변경, 모른다면 기본으로 말하기.
만약 대표 이름이 설정되지 않은 상태에서 구문 학습으로 이름이 처음으로 등장했다면 거기서 등장한 이름을 대표 이름으로 삼기.
- "파란색 인형"이라고 입력된 '파랑', '인형' 카드 조합에서 '인형' 카드에만 이름이 저장되어 있었다면 '파랑' 카드의 이름은 나머지 이름인 "파랑색"으로.

----
카드의 구현
카드 분야별로 나누어져 있는 카드들은 각각의 일련번호(?)가 있음 - 탈것 코드가 1이라면 1-1, 1-2...
카드 폴더 안에 분류별로 이미지들을 넣고, 그 폴더 내의 분류 폴더들을 순회하며 카드 이미지를 얻어낸다.
카드를 게임 상에 구현하는 건 CardCanvas에서 하고, 데이터적으로 다루는 건 GameManager가 한다.
내부적인 이름 구조는 Card_1_1이기 때문에, GameManager에서 다룰 때는 이 이름을 이용해 어떤 카드인지 알아낸다.

다중 카드 선택 시의 UI 구조
카드를 선택하면 selectedCards에 저장한다.
텍스트 창에서 우클릭을 누르면 카드 버리기.
Tab을 누르면 카드를 숨겨서 다른 카드를 추가할 수 있다.
